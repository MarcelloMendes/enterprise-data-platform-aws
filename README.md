# 📖 Overview

This project demonstrates the implementation of a complete **Serverless Enterprise Data Platform** on AWS.

The solution performs automatic ingestion, cataloging, transformation, querying and visualization of customer data using native AWS services.

The entire infrastructure was modeled using **Terraform**, following Infrastructure as Code (IaC) best practices.

---

# 🎯 Project Objectives

- Build a modern Data Lake architecture
- Automate data ingestion
- Catalog data automatically
- Process datasets using ETL
- Query data without databases
- Build Business Intelligence dashboards
- Provision infrastructure with Terraform

# ⚡ AWS Lambda

Lambda automatically starts the ingestion process whenever a new CSV file is uploaded to the Landing layer.

Responsibilities:

- Validate files
- Organize ingestion
- Trigger pipeline

Athena performs SQL queries directly over files stored in Amazon S3 without requiring a traditional database.

# ▶ Running the Project

## Clone repository

```bash
git clone https://github.com/MarcelloMendes/enterprise-data-platform-aws.git
```

---

## Initialize Terraform

```bash
terraform init

terraform validate

terraform plan
```

---

## Upload CSV

Upload datasets into:

```
landing/
```

---

## Execute Glue Job

```
etl-process-clientes
```

---

## Query with Athena

```sql
SELECT *

FROM enterprise_data_platform.raw_raw

LIMIT 10;
```

---

## Open Power BI

Open:

```
powerbi/EnterpriseDataPlatform.pbix
```

Refresh the data.
